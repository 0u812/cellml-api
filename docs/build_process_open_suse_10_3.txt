Build the CellML DOM API from its source code under openSUSE 10.3
=================================================================

Prerequisites
-------------

The Mozilla framework will have to be built first, if you are to use the CellML DOM API from within PCEnv.


Required packages
-----------------

- autoconf		A GNU tool for automatically configuring source code
- automake		A program for automatically generating GNU-style Makefile.in files
- gcc-c++		The system GNU C++ Compiler
- gsl-devel		GNU scientific library - development files
- libcppunit-devel	Include files and libraries mandatory for development
- libxml2-devel		Include files and libraries mandatory for development
- python-devel		Include files and libraries mandatory for building Python modules
- subversion		A concurrent versioning system similar to but better than CVS


omniORB
-------

omniORB is required to build the CellML DOM API, but it has to be installed separately from the required packages above.

omniORB 4.1.1 can be downloaded from http://sourceforge.net/project/showfiles.php?group_id=51138&package_id=44914&release_id=544978. You want omniORB-4.1.1.tar.gz. Then from a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
tar -xvf omniORB-4.1.1.tar.gz
rm omniORB-4.1.1.tar.gz
cd omniORB-4.1.1
mkdir build
cd build
../configure
make
sudo make install
cd ~
rm -r omniORB-4.1.1
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


Check out the source code
-------------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
svn co https://svn.physiomeproject.org/svn/physiome/CellML_DOM_API/trunk/ CellML_DOM_API
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Note: you will be told that the certificate is invalid, so you may want to ignore that message and accept the certificate permanently.


Build the CellML DOM API
------------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/CellML_DOM_API
aclocal && autoconf && automake
./configure --enable-context --enable-annotools --enable-cuses --enable-cevas --enable-malaes --enable-ccgs --enable-cis --enable-vacss
make
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Note #1: "aclocal && autoconf && automake" is not required if building from a source tar.
Note #2: if the CellML DOM API is to be used within PCEnv, then add "--enable-xpcom=/home/[Username]/mozilla-obj/obj-i686-pc-linux-gnu/stablexr/dist" to the "configure" command line.
Note #3: if the examples are required (e.g. CellML2C, RunCellML and ValidateCellML), then add "--enable-examples=yes" to the "configure" command line.


Test the CellML API
-------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/CellML_DOM_API
make check
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
