Download and uncompress the XUL Runner SDK (only if the CellML API is to be used within OpenCell)
=================================================================================================

The latest release of the XUL Runner SDK can be downloaded from http://releases.mozilla.org/pub/mozilla.org/xulrunner/releases/, e.g. http://releases.mozilla.org/pub/mozilla.org/xulrunner/releases/1.9.1.3/sdk/xulrunner-1.9.1.3.en-US.linux-i686.sdk.tar.bz2. From there, do the following to make it available in /Users/[Username]/xulrunner-sdk:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
tar -xvf xulrunner-1.9.1.3.en-US.linux-i686.sdk.tar.bz2
rm xulrunner-1.9.1.3.en-US.linux-i686.sdk.tar.bz2
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


The following other packages must be installed
==============================================

- XCode
- MacPorts       This can be installed from www.macports.org
- fink           This can be built from source; source and instructions are available at www.finkproject.org
- omniORB        This can be built from source; source and instructions are available at omniorb.sourceforge.net
- flex 2.5.33    This version must be built from source; source and instructions are available at flex.sourceforge.net

- wget           This can be installed through fink
- mercurial      This can be installed through fink
- autoconf       This can be installed through MacPorts
- automake       This can be installed through MacPorts
- libidl         This can be installed through MacPorts
- cppunit        This can be installed through MacPorts
- bison          This can be installed through MacPorts


Build the CellML DOM API from its source code
=============================================

When building the CellML DOM API and OpenCell, you must add to the configure command line CFLAGS="-arch i386" CXXFLAGS="-arch i386"


Check out the source code
-------------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
hg clone http://cellml-api.hg.sourceforge.net:8000/hgroot/cellml-api/cellml-api cellml
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

or the following, if you want to be able to push things to the official Mercurial repository:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
hg clone ssh://[Username]@cellml-api.hg.sourceforge.net/hgroot/cellml-api/cellml-api cellml
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


Java (only required if you want the Java binding to the CellML DOM API)
-----------------------------------------------------------------------

Java is (obviously) required to build the Java binding to the CellML DOM API.

Java SE Development Kit 6 Update 16 can be downloaded from http://java.sun.com/javase/downloads/index.jsp. You need to click on the "Download" button under the "Java SE Development Kit (JDK)" section. From there, select "Linux" as the platform, tick the "I agree to the Java SE Development Kit 6u16 License Agreement" checkbox, and click on the "Continue" button. A new page will open, giving you access to the download itself: jdk-6u16-linux-i586.bin. Once downloaded, do the following from a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
sudo mv jdk-6u16-linux-i586.bin /usr/local/share
cd /usr/local/share
chmod +x jdk-6u16-linux-i586.bin
sudo ./jdk-6u16-linux-i586.bin
sudo rm jdk-6u16-linux-i586.bin
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


Build the CellML DOM API
------------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/cellml
aclocal && autoconf && automake
./configure --enable-context --enable-annotools --enable-cuses --enable-cevas --enable-malaes --enable-ccgs --enable-cis --enable-vacss --enable-rdf --enable-celeds
make
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Note #1: "aclocal && autoconf && automake" is not required if building from a source tar.
Note #2: if the CellML DOM API is to be used within OpenCell, then add "--enable-xpcom=/Users/[Username]/xulrunner-sdk" to the "configure" command line.
Note #3: if the examples are required (e.g. CellML2C, RunCellML and ValidateCellML), then add "--enable-examples=yes" to the "configure" command line. 
Note #4: if you want the Java binding to the CellML DOM API, then add '--enable-java CPPFLAGS="-I/usr/local/share/jdk1.6.0_16/include -I/usr/local/share/jdk1.6.0_16/include/linux' to the "configure" command line.


Test the CellML API
-------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/cellml
make check
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


Install the CellML API
----------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/cellml
sudo make install
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
